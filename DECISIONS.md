# 결정 기록 (왜 이렇게 되어 있나)

> 코드만 봐서는 알 수 없는 **"왜"** 를 기록합니다.
> 같은 논의를 반복하거나, 이미 폐기한 방향으로 되돌아가지 않기 위함.

---

## 결제

### 왜 Toss가 아니라 포트원(PortOne)인가
- 처음엔 Toss Payments로 구현했음
- **Toss 직접계약: 가입비 22만 + 연관리비 11만 + 카드 3.4%**
- **포트원 추천패키지: 가입비·연관리비 전액 면제**, 월 거래 5,000만원 미만이면 포트원도 무료
- → 첫해 33만 + 매년 11만 절약. 코드는 다시 짰음 (2026-07-07)

### 왜 토스페이 결제수단이 없나
포트원에서 **카카오페이만 계약**됨 (토스페이는 채널에 없었음). 앱에서도 제거.

### 왜 가상계좌가 없나
**1,000원짜리 상품에 가상계좌는 건당 300원 정액** → 실질 수수료 30%.
정률(카드 ~3%)이 압도적으로 유리해서 의도적으로 제외.

### 왜 서버가 금액을 다시 계산하나
클라이언트가 `orders.amount`를 직접 쓰기 때문에, **Supabase REST를 직접 호출해 `amount: 1`로 주문 생성 후 1원 결제**가 가능했음(실제 취약점이었음).
→ 서버가 `zone/width/height/days/border_effect`로 **독립 계산**해서 대조하도록 수정.

## 격자 / 가격

### 왜 400×200(80,000칸)인가
원래 200×100(20,000칸)이었으나, **1칸이 20×20 실제픽셀(400px)로 너무 컸음**.
4등분해서 **1칸 = 10×10 픽셀(100px)** 로 변경. 지도 전체 크기는 그대로.
→ DB 재시딩 필요했음 (주소 체계가 `row*200+col` → `row*400+col`로 바뀜)

### 왜 칸당 1,000원 그대로인가
격자를 4배로 쪼갰지만 **가격은 유지** = 면적당 단가 4배. 사용자 결정.
(대안이던 "250원/칸으로 면적당 단가 유지"는 채택 안 함)

### 이용기간은 영구다 (기간제 아님)
`DURATIONS` 상수가 남아 있어 기간제로 오해하기 쉬우나, **선택 UI가 없고 `days`는 `PERMANENT_DAYS`로 하드코딩**.
모든 구매가 영구. `DURATIONS`는 **죽은 코드**.
※ 과거 약관/FAQ에 "기간 만료 후 갱신 안 하면 공간이 비워집니다"라는 **허위 안내**가 있었음 → 수정 완료.

### 왜 "기부" 표현을 다 뺐나
초기엔 "수익 일부를 디지털 문화 기금에 기부", "누적 기부금", "기부 증서"로 표기했으나,
**실제 기부처도 없고 기부도 하지 않으면서 기부를 표시 = 허위·과장광고**(표시광고법·전자상거래법)라
오픈 전 전부 제거함 (2026-07). 대안:
- "누적 기부금" → "입주한 집(채)"
- "기부 증서" → "소유 증서" (기능은 유지)
- FAQ의 기부 관련 Q&A 삭제
※ 나중에 실제 기부를 도입하려면 기부처·비율·정산을 실제로 갖춘 뒤에만 문구 복구할 것.

## 법적

### 왜 사업자정보 푸터를 못 지우나
전자상거래법 제10조 **필수 표기**. 게다가 **PG 심사가 이 정보를 크롤링해서 통과**했음.
→ 삭제 대신 **접기(toggle)**. 단 **DOM에는 항상 존재**해야 함 (`visibility`로만 숨김).

### 왜 결제 전 동의 체크박스 3개가 있나
전자상거래법 시행령 제21조 — 구매 확정 전 약관·환불·연령 동의 수집 **의무**.
없으면 위법. 미동의 시 결제 버튼 `disabled`.

### AI 콘텐츠 검사 — 사장님 확정 (2026-07)
사장님이 명시적으로 결정한 설정 (그 전엔 Claude가 기본값으로 켰던 것):
- **검사: 켬** — 결제 전 AI가 이미지+텍스트 검사, 통과해야만 결제 진행
- **모델: `claude-opus-4-8`** (건당 ~15원, 애매한 것도 잘 판별). 저비용 원하면 `check-content/route.ts`의 `MODEL`을 `claude-haiku-4-5`로
- **실패 시: 차단(fail-closed)** — 크레딧 소진·API 장애 시 입주를 막음 (검사를 일부러 실패시켜 우회하는 것 방지)
  - ⚠️ **운영 필수: Anthropic 콘솔에서 크레딧 잔액 알림 설정.** 안 그러면 크레딧 0 → 아무도 입주 못 함
- 호출 지점: `handlePayment`(신규, 결제 전) + `handleEditSave`(수정 시 재검사)

### 유해물 검열 — 현재 상태와 법적 판단
- **AI 검사는 가짜** (`setTimeout(2500)` 후 무조건 통과) → 지금은 무필터
- **법은 완벽한 사전검열을 요구하지 않음.** 핵심은 **신고 → 신속한 임시조치** (정통망법 제44조의2)
- 신고 기능 + `/admin` 신고탭 + 강제퇴거는 **이미 구현** → **법적 최소 요건은 충족**
- **남은 구멍:** 검사를 붙여도 **"수정"으로 우회 가능** (깨끗한 이미지로 통과 후 교체)
  → **수정 시 재검사**가 정답. `ANTHROPIC_API_KEY` 확보 시 구현

## 인프라

### 왜 sessionStorage를 안 쓰나
결제 리다이렉트 시 **모바일 카카오페이에서 sessionStorage가 유실**됨.
→ 주문 정보를 **`orders` 테이블에 저장**하고 서버가 조회하도록 변경.

### 비공개(is_visible) — 왜 "완전 비밀"이 아닌가
비공개 집도 지도 fetch(`page.tsx`)가 name·소개글·이미지까지 다 받아온 뒤 **UI에서만 가림**.
→ 개발자도구 네트워크 탭에서 비공개 집 내용이 보임 (클라이언트가 anon 키로 직접 조회하기 때문).
**완전 차단하려면** Supabase에 컬럼 마스킹 뷰/RPC(security definer) + 직접 SELECT 제한 RLS가 필요함.
그런데 그건 DB 수술이라 잘못 하면 지도가 안 뜨거나 더 샐 위험 → **테스트 환경 없이 라이브에 적용은 안 함.**
**해결 완료 (2026-07):** `public_houses` 마스킹 뷰 도입 (`supabase/public_houses_view.sql`).
- 비공개 집의 name/nickname/description/link_url/이미지를 소유자(auth.uid) 외엔 null 로 마스킹
- 지도 fetch(page.tsx) + 최근입주자(StatsPanel) 가 houses 대신 이 뷰를 읽음 → F12에도 안 보임
- ⚠️ 뷰 컬럼을 바꾸려면 houses 스키마와 뷰(SQL 파일) 둘 다 맞춰야 함
- 관리자·본인집수정·실시간은 houses 직접 사용 유지 (변경 없음)

### 왜 관리자 강제퇴거가 서버 라우트인가
원래 클라이언트에서 anon 키로 직접 DB를 썼음 → **RLS 없으면 아무나 강제퇴거 가능**했음.
→ `/api/admin/vacate`로 옮기고 **서버에서 JWT + 이메일 검증**.

### 왜 `.range(0, 79999)`가 붙어 있나
**Supabase는 기본 1,000행만 반환**함. 안 붙이면 집이 조용히 사라짐.
격자 크기 바꾸면 이 숫자도 같이 올려야 함.

## 겪은 버그 (재발 방지)

| 증상 | 원인 |
|---|---|
| 화면 전체가 뿌옇게 보임 | `ctx.scale(2,2)`에서 `lineWidth 0.4` → 0.8px 선이 픽셀 경계에 걸쳐 안티에일리어싱. **1px + 0.5 오프셋**으로 해결 |
| 최소 배율에서 지도 이동 불가 | `clampOffset`이 최소 스케일일 때 두 축 모두 중앙 고정. 하지만 최소 스케일은 cover 기준이라 **한 축은 항상 넘침** → 축별 독립 클램프로 수정 |
| 이미지 교체가 드래그해야 반영 | `imgReady`가 boolean이라 `setImgReady(true)` 재호출이 **no-op → 리렌더 안 됨**. 이미지 객체를 state로 변경 |
| 약관 페이지 스크롤 안 됨 | `globals.css`가 `body { overflow: hidden }`인데 페이지가 `minHeight:100vh` → **자체 스크롤 필요** |
| Vercel이 최신 커밋을 배포 안 함 | GitHub 웹훅이 가끔 안 튐 → 빈 커밋으로 재트리거 |
